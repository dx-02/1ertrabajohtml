Bienvenido
Hyper Text Markup Language

HTML es un markup language, es decir, lenguaje de Hiper Marcado de texto. Lo que significa que esta escrito en codigos que puede leer una persona sin que sea necesario compilarlo primiero. No es un lenguaje de programación; es un lenguaje de marcado que define la estructura del contenido. HTML es una serie de elementos que se usan para encerrar diferentes partes del contenido para que se vean o comporten de determinada manera. Se encarga de la estructura de la paguína, textos, imagenes, links, botones. Todo lo que se ve en la paguína y podemos interactuar hacer click y demas esta puesto con html.

ETIQUETAS:

Cuando se escribe código en HTML, estás escribiendo etiquetas HTML. Como dijimos anteriormete HTML no se considera un lenguaje de programación, ya que no puede crear una funcionalidad dinamica. Sin embargo los usuarios web pueden crear y estructurar secciones, párrafos y enlaces usando elementos, etiquetas y atributos. Las etiquetas se utilizan par delimitar el inicio y el final de los elementos en el marcado. Existen 142 etiquetas HTML disponibles que nos permite la creacion de varios elementos.

Etiquetas que pueden omitir su cierre:

Sí, ¡hasta la etiqueta html> puede omitirse!, en algunos casos.

  • < /html>: puede omitirse si el elemento html> no es seguido inmediatamente por un comentario.
  • < /head>: puede omitirse si el elemento head> no es seguido inmediatamente por un espacio o un comentario.
  • < /body>: puede omitirse si el elemento body> no es seguido inmediatamente por un comentario.
  • < /li>: puede omitirse si el elemento li> es seguido inmediatamente por otro elemento li> o si no hay más contenido en el elemento padre.
  • < /dt>: puede omitirse si el elemento dt> va seguido inmediatamente por otro elemento dt> o un elemento dd>.
  • < />: puede omitirse si el elemento dd> es seguido inmediatamente por otro elemento dd> o un elemento dt>, o si no hay más contenido en el elemento padre.
  • < /p>: puede omitirse si el elemento p> es seguido inmediatamente por un elemento address, article, aside, blockquote, details, div, dl, fieldset, figcaption, figure, footer, form, h1, h2, h3, h4, h5, h6, header, hr, main, nav, ol, p, pre, section, table, o ul, o si no hay más contenido en el elemento primario y el elemento principal es un elemento HTML que no es elementoa, audio, del, ins, map, noscript, o video` o un elemento personalizado autónomo.
  • < /rt>: puede omitirse si al elemento rt le sigue inmediatamente un elemento rt o rp, o si no hay más contenido en el elemento padre.
  • < /rp>: puede omitirse si el elemento rp es seguido inmediatamente por un elemento rt o rp, o si no hay más contenido en el elemento padre.
  • < /optgroup>: se puede omitir si el elemento de optgroup es seguido inmediatamente por otro elemento optgroup o si no hay más contenido en el elemento principal.
  • < /option>: puede omitirse si el elemento de option es seguido inmediatamente por otro elemento de option, o si lo sigue inmediatamente un elemento optgroup, o si no hay más contenido en el elemento principal.
  • < : puede omitirse si al elementocolgroup` no le sigue inmediatamente un carácter de espacio o un comentario.
  • < /caption>: puede omitirse si el elemento caption no es seguido inmediatamente por un carácter de espacio o un comentario.
  • < /thead>: se puede omitirse si el thead es seguido inmediatamente por un elemento tbody o tfoot.
  • < /tbody>: se puede omitirsi el elemento tbody es seguido inmediatamente por un elemento tbody o tfoot, o si no hay más contenido en el elemento principal.
  • < /tfoot>: puede omitirse si no hay más contenido en el elemento padre.
  • < /tr>:puede omitirse si el elemento tr es seguido inmediatamente por otro elemento tr, o si no hay más contenido en el elemento padre.
  • < /td>: puede omitirse si el elemento td es seguido inmediatamente por un elemento td o th, o si no hay más contenido en el elemento padre.
  • < pre/th>: puede omitirse si el elemento th es seguido inmediatamente por un elemento td o th, o si no hay más contenido en el elemento principal.

Etiquetas que abren y no necesitan de cierre:

Este tipo de etiquetas se denominan elementos de tipo "void", o elementos vacios. Los elementos de tipo void son aquellos que no pueden tener ningún contenido (incluyendo texto), y que la especificación dice explícitamente que solo tienen una etiqueta de apertura, y que no se debe especificar una etiqueta de cierre. Los elementos tipo "void" son:area, base, br,col, embed, hr, img, input, link, menuitem, meta,param, source, track, wbr.

-Encabezados:

Se utilizan para definir titulos. La etiqueta para los encabezados es " h> /h>", estas tienen un número de jerarquia siendo " h1> /h1>" el titulo principal y " h6> /h6>" el de menor importancia. Se recomienda que exista solo un h1 por documento como buena practica de SEO, las etiquetas de la h2 a la h6 se pueden tener tantas como sea necesarias. Otros ejemplos:

-Etiquetas de contenido:

*Todas las listas.

  • < p>: define que debe mostrarse como un parrafo.
  • < hr>: representa un cambio temático entre párrafos. Suele representar una línea horizontal.
  • < pre>:indica que su contenido está preformateado y que este formato debe ser preservado.
  • < blockquote>: representa un contenido sitado desde otra fuente.
  • < figure>: establece una figura ilustrada dentro del documento.
  • < figcaption>: representa la leyenda de una figura.

-Contenedores:

  • < div>: representa un contenedor genérico sin ningún significado especial.
  • < section>: comienza una seccion.
  • < article>: indica que ahi un articulo dentro de una sección.

*es indiferente el odenden de article y section, ademas cualquiera de los dos puede contener al otro. Section tambíen espera el uso de un h2 en adelante. *div se utiliza para darle formato a un parrafo por ejemplo con css.
